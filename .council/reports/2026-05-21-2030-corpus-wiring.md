## Out-of-scope additional change

While verifying, the FIRST submit attempt produced `[TranslationExercise] gradeAnswer returned null` because `src/app/session/page.tsx` was still merging `MOCK_SENTENCES` into the client pool. The scheduler picked `mock-s1` ("I have a car") which the disk corpus doesn't know. The new null-guard correctly dropped the answer, but the session would have been unable to progress.

Fix applied within this task (single additional file, follows the same "stop using mocks" thread):

- Session pool is now `contentSentences` (disk) + `dbSentences` (Supabase) only. Mock fixtures stay in `src/lib/mock-sentences.ts` for unit tests but no production code imports them.
- After this change: session picks "A dog is lying in front of the door." from the real corpus, grader resolves, no null branch hit.
